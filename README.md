# Security Community Practice

目前仓库主要包含两部分内容：

- `成员代码`：不同成员提交的独立实践项目。
- `漏洞靶场`：用于安全学习与演示的 XSS 挑战靶场。

## 仓库结构

```text
Security-Community-Practice/
├─ README.md
├─ 成员代码/
│  ├─ 张笑然-AStar自动寻路算法/
│  ├─ 朱瑜杰-BYR-IP-Manager/
│  ├─ 李秉卓-leecode算法hot100部分答案/

```

## 项目一览

| 项目名称 | 项目简介 |
| --- | --- |
| 漏洞靶场 | 基于多关卡页面的 XSS 学习靶场，适合安全基础练习与演示。 |
| 张笑然-AStar自动寻路算法 | 控制台版 A* 自动寻路算法示例，支持障碍物地图与路径打印。 |

## 项目规范

仓库中的项目目录组织与内容说明如下：

- 成员项目按独立目录进行组织，通常采用 `成员代码/成员名-项目名/` 的命名形式。
- 项目目录中可包含 `README.md`、依赖文件、源码、资源文件与数据库脚本等内容。
- 仓库不包含可直接公开使用的个人敏感信息、账号密码、Cookie、Token 或生产环境配置。
- 部分项目目录中包含缓存文件、构建产物或日志文件，实际阅读与运行时以源码和文档为准。
- 涉及数据库、第三方 API 或云服务的项目，通常需要额外环境变量或外部服务支持。

## 注意事项

- 本仓库中的各子项目彼此独立，依赖和运行方式并不统一。
- `漏洞靶场` 仅用于授权前提下的教学、练习与安全研究场景。
- 各子项目的具体依赖、运行要求与实现细节，以对应目录中的文档和源码为准。
